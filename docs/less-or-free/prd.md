## 项目背景

软件开发者在日常工作中需要大量工具来支撑软件工程全流程，但往往面临以下问题：

1. **信息分散**：优质工具散落在各处，缺乏统一收录和分类的平台
2. **成本不透明**：难以快速判断一个工具是否免费、是否需要翻墙、是否开源
3. **缺乏社区反馈**：缺少轻量级的方式来了解其他开发者对工具的评价
4. **版本不可追溯**：工具列表的变更无法追溯和对比，无法了解平台收录了哪些新工具、移除了哪些旧工具

**Less or Free** 旨在构建一个收录全网高性价比/免费软件工程工具的 Web 平台，按软件工程阶段分类，通过徽章系统快速传达工具属性，通过版本快照支持变更追溯，通过社区互动（star/评论/反对）提供轻量级反馈机制。

## 功能需求

### 模块：前台浏览

**REQ-001：分类列表浏览**

- **描述**：前台按软件工程阶段分类展示已上架工具。分类包括：需求分析、设计、开发、测试、部署、运维、项目管理、通用工具。每个工具可属于多个分类。
- **验收标准**：
  1. 首页展示各分类入口卡片，点击进入对应分类列表页
  2. 分类列表页仅展示已上架（status=published）的工具
  3. 工具按分类正确归属，支持一个工具出现在多个分类下
  4. 分类列表页展示工具名称、徽章、star 数量

**REQ-002：工具大全**

- **描述**：工具大全页面展示所有工具（包含已下架），支持筛选、搜索和排序。已下架工具有明显的"已下架"标记。
- **验收标准**：
  1. 展示所有工具（published + unpublished），已下架工具显示"已下架"标记
  2. 支持按分类筛选
  3. 支持按徽章属性筛选（付费类型、大陆友好度、开源类型、维护状态）
  4. 支持关键词搜索（工具名称和介绍）
  5. 支持按最新、最多 Star、名称排序
  6. 分页展示，每页 20 条

**REQ-003：徽章系统**

- **描述**：每个工具展示 4 组徽章，以圆形 SVG 图标呈现，圆形底色按程度分为绿色（好）、黄色（中）、红色（差），中心显示对应符号，悬停显示 tooltip 文字说明。
- **验收标准**：
  1. 付费类型徽章：中心符号 ¥，绿色=完全免费，黄色=增值服务，红色=付费
  2. 大陆友好徽章：中心符号 🌐，绿色=无需翻墙，黄色=单独配置，红色=需要翻墙
  3. 开源类型徽章：中心符号 </>，绿色=开源，黄色=部分开源，红色=闭源
  4. 维护状态徽章：中心符号 ⚙，绿色=活跃维护，黄色=维护中，红色=已停更
  5. 悬停时显示 tooltip 说明文字（如"完全免费"）
  6. 徽章在工具卡片和工具详情页均展示

### 模块：社区互动

**REQ-004：Star 功能**

- **描述**：用户可对工具进行 star 或取消 star，无需登录，通过浏览器指纹识别用户。
- **验收标准**：
  1. 点击 star 按钮触发 star，再次点击取消 star
  2. 同一指纹对同一工具只能 star 一次，重复点击为取消
  3. star 数量实时更新（反规范化计数）
  4. 页面加载时查询当前指纹的 star 状态，按钮显示对应状态

**REQ-005：评论功能**

- **描述**：用户可对工具发表扁平评论，无需登录，通过浏览器指纹识别。
- **验收标准**：
  1. 评论输入框 + 提交按钮，提交时附带浏览器指纹
  2. 评论按时间倒序分页展示，每页 20 条
  3. 每条评论显示内容和相对时间（如"3天前"）
  4. 每指纹每天最多发表 10 条评论
  5. 空内容评论前端校验拦截
  6. 评论数量实时更新（反规范化计数）

**REQ-006：反对功能**

- **描述**：用户可对工具点击反对，纯展示不影响排序，后台可查看反对统计。
- **验收标准**：
  1. 点击反对按钮需二次确认（"确定要反对这个工具吗？"）
  2. 同一指纹对同一工具只能反对一次，再次点击为取消
  3. 反对数量仅展示在工具详情页，不影响列表排序
  4. 反对数量实时更新（反规范化计数）
  5. 后台管理面板可查看各工具的反对数量

**REQ-008：免登录交互**

- **描述**：前台所有功能无需注册登录，通过浏览器指纹 + IP 限频防滥用。
- **验收标准**：
  1. 浏览、搜索、筛选无需任何认证
  2. star/反对通过浏览器指纹去重，指纹存入 localStorage
  3. 评论通过浏览器指纹 + KV 限频（每指纹每天 10 条）
  4. 通用 API 限频：每 IP 每分钟 60 次

### 模块：版本快照

**REQ-007：版本快照与 Diff**

- **描述**：每次管理员发布变更时自动生成版本快照，用户可查看历史版本列表，并对比任意两个版本的差异。
- **验收标准**：
  1. 版本历史页展示所有快照列表（版本号、描述、时间）
  2. 支持选择任意两个版本进行 diff 对比
  3. Diff 结果分三类展示：🟢 新增工具、🔴 移除工具、🟡 变更工具
  4. 变更工具展开显示具体字段差异（如"付费类型：免费→增值服务"）
  5. 版本号自增，每次发布 +1

### 模块：工具详情

**REQ-014：工具详情页**

- **描述**：点击工具后跳转到工具详情页，展示完整信息。
- **验收标准**：
  1. 展示工具名称、链接（可点击跳转到工具主页）、介绍
  2. 展示 4 组徽章
  3. 展示 star 数量和 star 按钮
  4. 展示反对数量和反对按钮
  5. 展示评论区（输入框 + 评论列表）
  6. 已下架工具在详情页顶部显示"已下架"提示

### 模块：后台管理

**REQ-009：管理员登录**

- **描述**：管理员通过密码登录后台，使用 HttpOnly Secure Cookie 鉴权。
- **验收标准**：
  1. 登录页提供密码输入框和登录按钮
  2. 登录成功后设置 HttpOnly Secure Cookie，有效期 24 小时
  3. Cookie 中包含 admin_token，后台 API 通过校验 token 验证身份
  4. 未登录访问后台页面自动跳转登录页
  5. 支持登出功能

**REQ-010：工具管理（草稿→发布）**

- **描述**：管理员可新增/编辑工具，保存为草稿状态，支持统一发布。
- **验收标准**：
  1. 新增工具表单：名称、链接、介绍、4 组徽章选择、分类选择（多选）
  2. 保存后工具状态为 draft
  3. 编辑已有工具，修改后状态保持不变（已发布的仍为 published）
  4. 工具管理列表展示所有工具（含 draft），支持筛选和搜索
  5. 支持对工具进行上架（draft/unpublished → published）和下架（published → unpublished）操作

**REQ-011：一键发布**

- **描述**：管理员查看待发布变更后，一键发布并自动生成版本快照。
- **验收标准**：
  1. 发布管理页展示待发布的变更列表
  2. 管理员填写版本描述后点击发布
  3. 发布时将所有 status=published 的工具序列化为 JSON 存入快照表
  4. 快照版本号自增
  5. 发布成功后前台内容更新

**REQ-012：反对统计**

- **描述**：后台可查看各工具的反对数量，辅助上下架决策。
- **验收标准**：
  1. 管理面板展示高反对工具列表（按反对数降序）
  2. 快照管理页可查看各工具的反对统计
  3. 支持从统计页直接跳转到工具管理页进行上下架操作

**REQ-013：工具上下架**

- **描述**：管理员可对工具进行上架/下架操作。
- **验收标准**：
  1. 上架：工具状态变为 published，前台分类列表和工具大全可见
  2. 下架：工具状态变为 unpublished，前台分类列表不可见，工具大全可见但标记"已下架"
  3. 上下架操作需在下次发布时才生成快照记录

## 接口设计

### 统一响应格式

```typescript
type ApiResponse<T> = {
  code: number
  data?: T
  message?: string
}
```

| code | 含义 | data |
|------|------|------|
| 200 | 成功 | 响应数据 |
| 401 | 未登录 | 无 |
| 403 | 禁止访问 | 无 |
| 404 | 不存在 | 无 |
| 409 | 重复操作 | 无 |
| 429 | 限频 | 无 |
| 422 | 参数错误 | message 含具体错误 |
| 500 | 服务异常 | 无 |

### 前台接口

#### 接口：获取工具列表

- **路径**：`GET /api/tools`
- **描述**：获取工具列表，支持筛选、搜索、排序、分页
- **请求参数**：

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| category | string | 否 | 分类 slug 筛选 |
| badge | string | 否 | 徽章筛选（格式：badge_payment:free） |
| search | string | 否 | 关键词搜索（名称+介绍） |
| sort | string | 否 | latest / stars / name，默认 latest |
| status | string | 否 | published / all，默认 published |
| page | number | 否 | 页码，默认 1 |

- **响应**：

| 字段名 | 类型 | 描述 |
|--------|------|------|
| code | number | 200 |
| data.tools | array | 工具列表 |
| data.total | number | 总数 |
| data.page | number | 当前页码 |
| data.pageSize | number | 每页条数 |

#### 接口：获取工具详情

- **路径**：`GET /api/tools/[id]`
- **描述**：获取单个工具的完整信息
- **请求参数**：

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| id | string | 是 | 工具 ID（路径参数） |

- **响应**：

| 字段名 | 类型 | 描述 |
|--------|------|------|
| code | number | 200 |
| data | object | 工具完整信息（含分类列表） |

#### 接口：获取分类列表

- **路径**：`GET /api/categories`
- **描述**：获取所有分类及其工具数量
- **响应**：

| 字段名 | 类型 | 描述 |
|--------|------|------|
| code | number | 200 |
| data | array | 分类列表（含 toolCount） |

#### 接口：Star / 取消 Star

- **路径**：`POST /api/tools/[id]/star`
- **描述**：对工具 star 或取消 star
- **请求参数**：

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| id | string | 是 | 工具 ID（路径参数） |
| fingerprint | string | 是 | 浏览器指纹（body） |

- **响应**：

| 字段名 | 类型 | 描述 |
|--------|------|------|

## 风险与假设

### 风险

| 编号 | 风险描述 | 影响 | 概率 | 缓解措施 |
|------|----------|------|------|----------|
| R-001 | Cloudflare D1 仍处于 Beta 阶段，可能存在稳定性问题 | 中 | 低 | 数据定期导出备份；关键查询做好降级处理 |
| R-002 | Edge Runtime 不支持部分 Node.js API，可能影响开发 | 中 | 中 | 开发前确认 API 兼容性，使用 @cloudflare/next-on-pages 适配 |
| R-003 | 浏览器指纹可能被篡改或清除，导致交互数据异常 | 低 | 中 | 指纹仅用于轻量级去重，不作为核心安全依赖 |
| R-004 | 免登录评论可能被恶意灌水 | 中 | 中 | KV 限频（每指纹每天10条）+ 管理员后台可删除 |
| R-005 | 快照数据量随版本增长，D1 存储可能超限 | 中 | 低 | 快照 JSON 可压缩存储；定期清理过旧快照 |

### 假设

- 项目初期为单管理员模式，不需要多用户权限系统
- 浏览器指纹足以满足轻量级去重需求，不需要严格的用户身份体系
- Cloudflare D1 免费额度（5GB 存储 + 5M 读/100K 写每日）足够支撑项目初期
- 工具数量预期在数百级别，不会产生性能瓶颈
- Cloudflare KV 最终一致性模型对评论限频计数可接受
